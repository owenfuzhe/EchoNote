**版本**: v0.3

---

## 1. 产品定位

AI-Native 的笔记应用，核心理念是「第二大脑」—— 自动捕获、智能关联、主动洞察。

**新增：Context Graph 愿景**
EchoNote 不仅是存放结果的静态仓库，更是记录思考过程的动态引擎。通过捕获「决策痕迹（Decision Traces）」，让 AI 不仅知道"发生了什么"，更理解"为什么发生"，从而构建真正个性化的知识图谱。

### 3.2 功能模块

#### 3.2.1 晨间洞察 (P0)
- **触发**: 每日首次打开 App
- **内容**: AI 分析昨日笔记，发现与历史内容的关联
- **展示**: 卡片形式，可展开查看详情

#### 3.2.2 进行中的任务 (P0)
- **来源**: 用户标记为"进行中"的笔记
- **展示**: 进度条 + 实时状态更新
- **操作**: 点击进入详情，支持暂停/继续

#### 3.2.3 最近记忆 (P0)
- **数量**: 横向滚动，显示最近 3-5 条
- **内容**: 笔记缩略图 + 标题
- **交互**: 点击进入 DocumentView

#### 3.2.4 动态流 (P0)
- **内容类型**:
  - 💡 灵感（从语音/文字提取的关键想法）
  - 🎯 待办（从所有笔记聚合的待办事项）
  - 🔗 关联（AI 发现的笔记间联系）
  - ✨ 智能融合（跨系统意图捕获）
- **数据来源**: 实时从笔记内容提取
- **排序**: 时间倒序

**新增：跨系统意图捕获 (Context Graph P1)**
- **时间窗聚合**: 10 分钟内的多模态输入自动关联
- **实体融合**: 图片 OCR + 语音语义合并为完整记录
- **展示**: 动态流显示"AI 智能融合"卡片

## 7. Context Graph 系统（新增）

### 7.1 笔记溯源面板 (Context Trace Panel) P1
记录"Decision Traces"，解决"为什么会有这条笔记"的问题。

**功能描述：**
1. **隐式记录**: AI 生成内容时自动记录：
   - `Trigger`: 系统触发或用户手动触发
   - `Inputs`: 引用的所有源文件 ID
   - `Prompt/Overrides`: 用户干预的修改指令

2. **显性展示**: 单篇笔记底部折叠的"溯源 (Trace)"区域

**UI 草图：**
```
+------------------------------------------+
|  📄 竞品调研报告 (最终态)                 |
|  ... (正文内容) ...                       |
|                                          |
|  --- 🔽 展开溯源 (Context Trace) -------- |
|  [ 🧠 生成于 2026-03-01 14:00 ]           |
|  基于以下 3 个上下文缝合：                |
|  1. 🔗 [网页剪藏] A公司定价策略            |
|  2. 🎙️ [语音碎片] "重点看A公司的企业版"     |
|  3. 💬 [用户干预] "帮我删掉技术架构对比"    |
|                                          |
|  [ ⏪ 重放生成过程 ]  [ 🔄 添加新信息重写 ] |
+------------------------------------------+
```

### 7.2 动态先例引擎 (Dynamic Precedent Engine) P2
记录用户的"例外与偏好"，让 AI 越用越顺手。

**功能描述：**
1. **行为捕获**: 监听用户对 AI 生成内容的修改（>30% 结构性删改）
2. **图谱更新**: AI 静默提取"变更意图"写入用户偏好节点
3. **用户感知**: 轻量级 Toast 提示：
   > ✨ "已将您的先例记入图谱：未来的科技类播客将默认关闭背景音乐。 [撤销]"

### 7.3 数据模型扩展

```typescript
// 决策痕迹
interface ContextTrace {
  id: string;
  noteId: string;
  trigger: 'manual' | 'scheduled' | 'ai_suggested';
  inputs: string[]; // 源文件 ID 列表
  prompt?: string; // 用户干预指令
  generatedAt: string;
  model: string; // 使用的 AI 模型
}

// 用户先例偏好
interface UserPrecedent {
  id: string;
  category: 'format' | 'tone' | 'structure' | 'voice';
  pattern: string; // 模式描述
  frequency: number; // 触发次数
  lastUsed: string;
}

// 跨系统关联
interface CrossSourceLink {
  id: string;
  sourceIds: string[]; // 关联的碎片 ID
  type: 'time_window' | 'semantic' | 'manual';
  confidence: number;
  createdAt: string;
}
```

## 8. 技术实现要点

### 8.1 优先级划分（更新）

| 优先级 | 功能 |
|--------|------|
| P0 | 全局捕获枢纽、Home 四大模块、Library 主题聚类、联网搜索、播客生成、语音捕获 |
