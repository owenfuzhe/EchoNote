## 1. 产品概述

### 1.1 产品名称
EchoNote - AI 驱动的语音笔记应用

### 1.2 产品定位
一个集成了语音识别和 AI 智能处理的笔记应用，让用户可以通过语音快速记录想法，AI 自动整理、分类和生成结构化内容。

### 1.3 核心价值主张
- 🎙️ **语音优先**：快速语音输入，解放双手
- 🤖 **AI 智能处理**：自动转录、摘要、标签分类
- 📝 **结构化输出**：语音 → 结构化的可搜索笔记
- 🔍 **智能检索**：语义搜索，快速找到所需内容

---

## 2. 目标用户

### 2.1 主要用户画像
1. **忙碌的专业人士**：会议记录、灵感捕捉
2. **内容创作者**：播客、视频脚本草稿
3. **学生**：课堂笔记、学习总结
4. **INTP 类型用户**：想法多、需要快速记录和整理

---

## 3. 功能需求

### 3.1 MVP 核心功能（第一阶段）

#### 3.1.1 语音录制与转录
- [ ] 一键开始/停止录音
- [ ] 实时语音转文字（支持中文/英文）
- [ ] 录音本地存储 + 云端备份

#### 3.1.2 AI 智能处理
- [ ] 自动摘要生成（TL;DR）
- [ ] 自动标签分类（#工作 #灵感 #待办等）
- [ ] 关键信息提取（日期、地点、人物、任务）

#### 3.1.3 笔记管理
- [ ] 笔记列表视图（按时间/标签筛选）
- [ ] 搜索功能（关键词 + 语义搜索）
- [ ] 笔记编辑（支持修改转录文本）

#### 3.1.4 导出与分享
- [ ] 导出为 Markdown / TXT / PDF
- [ ] 分享链接生成

### 3.2 进阶功能（第二阶段）

- [ ] 多语言支持
- [ ] 语音指令（"标记为重要"、"创建待办"）
- [ ] 与其他工具集成（Notion、Obsidian）
- [ ] 协作功能（共享笔记本）

## 4. 技术架构

### 4.1 技术栈（建议）
- **前端**：Next.js + Tailwind CSS
- **后端**：Python FastAPI
- **数据库**：PostgreSQL + Redis
- **语音识别**：Whisper API / 本地 Whisper
- **AI 处理**：OpenAI GPT / Claude / 国产大模型

### 4.2 部署架构
- 前端：Vercel / Netlify
- 后端：VPS / Railway / Fly.io
- 数据库：Supabase / 自托管

---

## 5. UI/UX 设计原则

### 5.1 设计关键词
- **极简**：界面干净，无干扰
- **快速**：3秒内开始录音
- **智能**：AI 处理无缝融入工作流

### 5.2 核心页面
1. **首页/录音页**：大录音按钮 + 最近笔记
2. **笔记详情页**：转录文本 + AI 摘要 + 编辑功能
3. **笔记列表页**：卡片式布局 + 筛选器
4. **设置页**：语言、导出格式、AI 模型选择

## 7. 已决策事项 ✅

| 决策项 | 方案 | 理由 |
|--------|------|------|
| **语音识别** | 端侧原生能力（本地处理） | 隐私优先，离线可用 |
| **AI 处理模型** | Kimi / GLM（国产大模型） | 合规，成本可控 |
| **数据存储** | 默认云端全量 | 体验优先，多端同步 |
| **商业模式** | 免费 + 付费增值 | 基础功能免费，AI高级功能付费 |
