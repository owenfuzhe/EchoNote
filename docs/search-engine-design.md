# EchoNote 智能搜索引擎设计方案

## 需求背景

当前搜索页面使用简单的字符串匹配，需要升级为内置的智能搜索引擎，支持：
- 全文检索（标题 + 内容）
- 模糊搜索（拼写容错）
- 相关性排序
- 搜索结果高亮
- 搜索建议/补全

## 方案对比

### 方案一：Fuse.js（推荐）

**特点：**
- 轻量级（~10KB gzip）
- 模糊搜索（基于 Bitap 算法）
- 支持权重配置（标题权重 > 内容权重）
- 支持距离阈值配置
- 无需构建索引，即插即用

**优点：**
- 实现简单，零配置即可使用
- 对拼写错误容错性好
- 支持中文搜索
- 性能在千级数据量下表现良好

**缺点：**
- 万级以上数据量性能下降
- 不支持复杂的布尔查询

**适用场景：**
- 个人笔记应用，数据量 < 10,000 条
- 需要快速实现，不依赖后端

### 方案二：Lunr.js

**特点：**
- 基于倒排索引
- 支持更复杂的查询语法（AND/OR/NOT）
- 支持字段权重配置
- 支持搜索词高亮

**优点：**
- 搜索性能更好（有索引）
- 支持布尔查询
- 可配置性强

**缺点：**
- 需要构建索引（初始化时间）
- 包体积较大（~20KB）
- 中文支持需要额外配置

**适用场景：**
- 数据量较大（> 10,000 条）
- 需要复杂查询语法

### 方案三：Minisearch

**特点：**
- 结合 Fuse.js 的易用性和 Lunr 的性能
- 自动索引
- 支持模糊匹配
- 支持字段权重

**优点：**
- 平衡了易用性和性能
- 支持自动建议
- 体积小（~6KB）

**缺点：**
- 社区相对较小

## 推荐方案：Fuse.js + 自定义优化

### 实现架构

```
┌─────────────────────────────────────┐
│           SearchService             │
├─────────────────────────────────────┤
│  - 初始化 Fuse 实例                  │
│  - 管理搜索索引                      │
│  - 提供搜索 API                      │
│  - 结果高亮处理                      │
└─────────────────────────────────────┘
                  │
                  ▼
┌─────────────────────────────────────┐
│           Fuse.js 引擎              │
├─────────────────────────────────────┤
│  - 模糊匹配算法                      │
│  - 权重计算                          │
│  - 结果排序                          │
└─────────────────────────────────────┘
```

### 配置参数

```typescript
const fuseOptions = {
  // 搜索字段及权重
  keys: [
    { name: 'title', weight: 0.6 },      // 标题权重最高
    { name: 'content', weight: 0.3 },    // 内容次之
    { name: 'tags', weight: 0.1 }        // 标签辅助
  ],
  // 模糊匹配阈值：0.0 = 完全匹配, 1.0 = 匹配所有
  threshold: 0.4,
  // 距离阈值（用于长文本）
  distance: 100,
  // 是否包含匹配位置信息（用于高亮）
  includeMatches: true,
  // 忽略位置（提升性能）
  ignoreLocation: true,
  // 最小匹配字符数
  minMatchCharLength: 2
};
```

### 搜索功能特性

1. **智能高亮**
   - 使用 `includeMatches` 获取匹配位置
   - 对匹配文本添加高亮样式

2. **搜索建议**
   - 基于历史搜索记录
   - 基于笔记标题关键词提取

3. **结果排序**
   - 按相关性分数排序
   - 支持时间/相关性混合排序

4. **增量搜索**
   - 输入时实时搜索（防抖 200ms）
   - 结果即时更新

### API 设计

```typescript
interface SearchResult {
  item: Note;
  score: number;
  matches: Array<{
    key: string;
    value: string;
    indices: Array<[number, number]>;
  }>;
}

class SearchService {
  // 初始化/更新索引
  init(notes: Note[]): void;
  
  // 执行搜索
  search(query: string): SearchResult[];
  
  // 获取搜索建议
  getSuggestions(query: string): string[];
  
  // 高亮匹配文本
  highlight(text: string, matches: Match[]): ReactNode;
}
```

## 实施步骤

1. **安装依赖**
   ```bash
   npm install fuse.js
   ```

2. **创建 SearchService**
   - 路径：`src/services/search-engine.ts`
   - 封装 Fuse.js 实例
   - 提供高亮工具函数

3. **更新 SearchView 组件**
   - 集成 SearchService
   - 实现实时搜索
   - 添加结果高亮显示

4. **优化搜索 UI**
   - 搜索框样式优化
   - 结果列表展示优化
   - 空状态提示

## 性能考虑

- **索引更新**：笔记增删改时增量更新索引
- **防抖处理**：输入搜索时 200ms 防抖
- **虚拟滚动**：搜索结果过多时使用虚拟滚动
- **Web Worker**：大数据量时考虑使用 Web Worker

## 后续扩展

1. **AI 语义搜索**：接入 Embedding 模型，支持语义相似搜索

3. **高级筛选**：按时间、标签、类型筛选搜索结果
4. **搜索统计**：分析搜索热词，优化笔记组织
