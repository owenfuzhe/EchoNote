> 版本：v1.0

## 📱 设计概述

首页是 EchoNote 的核心页面，用户在这里录制语音笔记、查看最近记录、导航到其他功能。设计采用移动端优先策略。

### 设计原则
- **简洁直观**：核心功能（录音）一眼可见
- **单手操作友好**：所有主要操作在拇指可及范围内
- **即时反馈**：录音状态清晰可见
- **快速访问**：最近笔记一目了然

## 🎭 关键交互说明

### 1. 开始录音

| 触发 | 用户点击大录音按钮 |
|------|-------------------|
| 反馈 | 按钮缩放动画 → 波形区激活 → 开始计时 |
| 状态变化 | 空闲 → 录音中 |
| 特殊 | 底部最近笔记列表半透明化，减少干扰 |

**动画细节：**
- 按钮点击：scale 0.95 → 1.0，200ms 弹性动画
- 波形区：从静态图标平滑过渡到动态波形
- 标题：淡出淡入切换文字

### 2. 录音中操作

| 操作 | 触发 | 反馈 |
|------|------|------|
| **停止** | 点击左按钮 | 弹出保存对话框 |
| **暂停** | 点击中按钮 | 波形冻结，时长暂停，按钮变为"继续" |
| **取消** | 点击右按钮 | 二次确认弹窗，确认后丢弃录音 |

**波形动画：**
- 频率：实时响应麦克风输入
- 样式：渐变色波形 (蓝色→紫色)
- 峰值保持：短暂显示最高音量条

### 3. 暂停/继续

| 触发 | 点击暂停/继续按钮 |
|------|-------------------|
| 状态 | 录音中 ↔ 暂停 |
| 视觉 | 波形冻结/恢复，时长旁显示 ⏸️/▶️ |
| 逻辑 | 暂停时不写入文件，继续时追加 |

### 4. 保存录音

| 步骤 | 操作 |
|------|------|
| 1 | 点击停止按钮 |
| 2 | 弹出保存对话框，默认标题"新录音"+时间戳 |
| 3 | 用户可编辑标题 |
| 4 | 点击保存 → 文件保存 → 列表刷新 |
| 5 | 点击取消 → 询问"是否丢弃" → 确认后删除 |

### 5. 最近笔记交互

| 操作 | 触发 | 结果 |
|------|------|------|
| **查看** | 点击笔记项 | 跳转到笔记详情页 |
| **播放** | 点击播放图标 | 内嵌播放该录音 |
| **更多** | 长按/右滑 | 显示操作菜单（重命名、删除、分享） |
| **刷新** | 下拉列表 | 刷新最近笔记列表 |

### 6. 底部导航

| 图标 | 页面 | 说明 |
|------|------|------|
| 🏠 首页 | 当前页 | 录音+最近笔记 |
| 📝 笔记 | 笔记列表页 | 所有笔记的完整列表 |
| ⚙️ 设置 | 设置页 | 应用配置 |

**交互细节：**
- 当前页面图标高亮（主色）
- 切换页面时：淡入淡出过渡
- 录音中切换页面：提示"正在录音，确定离开？"

## 📝 待办/注意事项

### 技术实现提示
1. **音频权限**：首次进入页面时请求麦克风权限
2. **后台录制**：支持锁屏后继续录制
3. **打断处理**：来电时自动暂停，恢复后继续
4. **存储空间**：录制前检查剩余空间，不足时提示
5. **自动保存**：意外退出时自动保存为草稿

### 性能优化
- 波形使用 Canvas 渲染，避免 DOM 操作
- 录音数据分段写入，减少内存占用
- 列表使用虚拟滚动，处理大量笔记

### 可访问性
- 按钮添加 contentDescription
- 支持 VoiceOver/TalkBack
- 颜色对比度符合 WCAG AA 标准
